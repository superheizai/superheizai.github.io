我查看了我的样式与zipperary的样式的差别，发现了一些端倪。我的这里关于这个样式的定义如下:

.article-entry blockquote {
font-family: Georgia, “Times New Roman”, serif;
font-size: 1.4em;
margin: 1.6em 20px;
text-align: center;
}

这里的text-align是center，居中。而在zipperary的样式中，它引用到了另外一种截然不同的样式:

font-family: Georgia, ‘Times New Roman’, Times, STFangsong, KaiTi, serif;
font-size: 1.2em;
padding: 0 30px 15px;
padding-top: 10px;
padding-bottom: 30px;
}

logrotate日志处理

概要

logrotate是日志处理脚本，可以按照时间维度，天，日，月或者年来执行，也可以按照文件大小来执行。
logrotate是通过cron定时任务来执行的，其入口文件在/etc/logrotate.conf里面.在centos上可以查看/etc下面cron相关配置中，可以看到相关的配置。

在anacrontab中定义了执行cron.daily下面的所有文件(run-parts)
run-parts /etc/cron.daily
在/etc/cron.daily文件夹下面，有logrotate文件

#!/bin/sh

/usr/sbin/logrotate /etc/logrotate.conf  >/dev/null 2 >&1
EXITVALUE=$?
if [ $EXITVALUE != 0 ]; then
/usr/bin/logger -t logrotate "ALERT exited abnormally with         [$EXITVALUE]"
fi
exit 0

cut命令使用

之前，我们针对行，针对分隔符，抽取某列数据，一般会用awk命令

今天，突然发现对于抽取某一列的情况可以使用cut命令解决这个问题

但是，cut的使用场景是有限制的
